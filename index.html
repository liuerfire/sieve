Anti-patterns: things to avoid

simonwillison.net | Wed, 04 Mar 2026 17:34:42 +0000
💡 深入探讨智能体工程中的反模式与工程实践，强调AI辅助编程下的代码验证、协作规范与开发者责任，高度契合编程效率、工程实践与技术深度解析等核心兴趣。

TL;DR: 在智能体工程（agentic engineering）中，将未经审查的AI生成代码直接提交给协作者是一种常见但有害的反模式（anti-pattern），开发者应确保代码功能正确、变更粒度合理，并提供充分上下文，以尊重评审者的时间与专业性。

  • 核心反模式：直接提交由编码智能体（coding agent）生成但未经本人验证的代码至 Pull Request（PR），实质上是将本应由自己完成的验证工作转嫁给团队成员。
  • 高质量 PR 的标准：代码必须经过开发者亲自验证其功能正确性；变更应足够小以便高效评审；需附带高层目标说明及关联问题或规范链接。
  • PR 描述亦需审核：AI 生成的 PR 描述虽具说服力，但开发者仍需亲自阅读并核实，不可直接转发未经检查的内容。
  • 提供验证证据：建议在 PR 中包含手动测试记录、关键实现决策注释，甚至功能演示截图或视频，以证明已投入必要精力，避免浪费评审者时间。

随着生成式 AI 和编码智能体在软件开发中的普及，团队协作流程面临新挑战。若开发者过度依赖 AI 而忽视自身责任，不仅降低代码质量，还侵蚀团队信任与效率。本文强调，在“智能体工程”时代，开发者的核心价值并非仅在于生成代码，而在于对代码的验证、拆解与上下文构建——这是维持工程严谨性和协作可持续性的关键。

💡 Did you know? “Agentic Engineering（智能体工程）”指利用具备自主规划与执行能力的 AI 智能体（如能调用工具、迭代修改代码的 LLM 应用）参与软件开发的新范式，其高效性也带来了新的工程伦理与协作规范挑战。

TL;DR: 开发者 Simon Willison 利用 HTTP Range 请求（HTTP range requests）和二分查找（binary search）技术，构建了一个高效的 Unicode 探索工具，可在不下载完整 76.6MB 数据文件的情况下快速查询字符信息。

  • 该工具通过浏览器中的 fetch() 发起 HTTP Range 请求，对托管在 S3 + Cloudflare 上的大型 Unicode 元数据文件执行二分查找，仅传输必要字节（如查询“&”仅需 3,864 字节）。
  • 为确保字节偏移准确，请求中显式禁用压缩（'Accept-Encoding': 'identity'），但实际因 CDN（如 Cloudflare）在检测到 Range 请求时自动跳过压缩，此设置非必需。
  • 项目由 AI 辅助开发：作者使用 Claude 大语言模型（LLM）进行需求构思、规范生成和代码实现，体现了“AI 辅助编程”（AI-assisted programming）与“异步研究”（asynchronous research）的工作流。
  • Unicode 数据文件按码点（codepoint）排序，天然适合二分查找，使单字符或十六进制码点（如 1F99C）查询可在约 17 步内完成。

这一实现展示了如何结合经典算法（二分查找）与现代 Web 技术（HTTP Range 请求）高效处理大型静态数据集，同时避免全量下载。在带宽受限或移动端场景下，此类技术可显著提升性能。此外，项目也反映了当前开发者如何利用生成式 AI（Generative AI）加速原型构建，将抽象想法快速转化为可运行工具。

💡 Did you know? HTTP Range 请求通常用于视频流或大文件分块下载，但也可用于随机访问结构化数据——只要文件按查询键（如 Unicode 码点）有序排列，就能支持高效的远程二分查找。
